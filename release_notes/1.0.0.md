## 1.0.0 - Initial Release

### Features

- **SOCRadar API v4**: Fetches incidents from the latest v4 endpoint with header-based API key authentication
- **Automatic Incident Ingestion**: Polls incidents with configurable intervals and state-based checkpointing
- **IOC Extraction**: Extracts IP, URL, domain, hash, and email indicators from `alarm_related_entities` and `content` fields into standard CEF format
- **Severity Mapping**: Maps `alarm_risk_level` (CRITICAL/HIGH/MEDIUM/LOW) to SOAR severity levels
- **State-Based Deduplication**: Tracks alarm status in persistent state to prevent duplicate containers
- **Status Change Detection**: Creates new artifacts when alarm status changes (OPEN, INVESTIGATING, RESOLVED, etc.)
- **Poll Now Support**: Manual polling without updating state checkpoint
- **Rate Limit Handling**: Reads `retry-after` and `x-ratelimit-remaining` response headers for smart throttling
- **Get Incident Action**: Retrieve details of a single SOCRadar alarm by ID

### Supported Actions

| Action | Type | Description |
|--------|------|-------------|
| **test connectivity** | test | Validates API credentials and network connectivity |
| **on poll** | ingest | Ingests SOCRadar incidents with deduplication |
| **get incident** | investigate | Retrieves a single incident by alarm ID |
| **update status** | generic | Changes the status of a SOCRadar alarm |
| **change severity** | generic | Changes the severity of a SOCRadar alarm |

### Configuration

| Parameter | Required | Default | Description |
|-----------|----------|---------|-------------|
| socradar_company_id | Yes | — | SOCRadar Company ID |
| socradar_api_key | Yes | — | SOCRadar API Key |
| verify_server_cert | No | false | Verify SSL certificates |
| first_run_max_incidents | No | 1000 | Max incidents on first run (last 10 hours) |
| max_incidents_per_poll | No | 100 | Max incidents per scheduled poll |

### Requirements

- Splunk SOAR 6.3.0 or later
- Python 3.9 or 3.13
- SOCRadar platform account with API access
